# 設計一款節奏遊戲"SUTO"
- 使用 Phaser 3（建議版本，穩定且文件完善）來開發
- 需要有進入遊戲的畫面
- 玩家使用滑鼠操作
- 每個檢定環節分兩個八拍，第一個八拍先顯示提示（提示階段），第二個八拍玩家需要根據提示進行操作（檢定階段）
- 遊戲用橫長方形螢幕顯示，在遊戲畫面上畫一個橢圓，這會是檢定出現的位置，為了 debug 方便，先畫一個白色橢圓線

## 定義方位編號
上: U
下: D
左: L
右: R
左上: UL
右上: UR
左下: DL
右下: DR


## 提示階段
八個圖示用 4x2 的方式排列，每個圖示可能是：U、D、L、R、UL、UR、DL、DR 其中一個
這些圖示來自同一張圖 `src/assets/down.png`，這張圖是往下的箭頭，要透過旋轉來達成不同方向的圖示，斜角的情況
`src/assets/down_left.png` 是左下的箭頭，其他斜角的圖示也是透過旋轉來達成
每一拍高亮當前拍對應的圖示，讓玩家清楚看出該拍要做的動作

- 特殊提示：旋轉，遇到這個提示時，4x2 張圖示全部同步繞自己的中心點旋轉，每一拍旋轉 90 度，玩家需要根據旋轉方向來進行檢定
  - 所有圖示同步顯示相同方向，一起旋轉
  - 旋轉動畫必須平滑，不能有卡頓
  - 旋轉點只能根據相鄰的兩個方向來決定，例如：U 只能轉到 UR 或 UL，不會跳到非相鄰方向
  - 旋轉是每拍必定經過兩個同方向的單位，例如：U 會先轉到 UR，再轉到 R
  - 一定會從 U、D、L、R 其中一個方向開始轉

## 檢定階段

### 檢查點 / 檢定線
- 不再把檢定位置畫在橢圓上；橢圓只保留 debug 參考線
- U、D、L、R：判定目標是距離遊戲畫面上、下、左、右邊界 `checkDepth` pixel 的檢定線
  - U：判定框碰到距離上邊界 `checkDepth` pixel 的水平線即成功
  - D：判定框碰到距離下邊界 `checkDepth` pixel 的水平線即成功
  - L：判定框碰到距離左邊界 `checkDepth` pixel 的垂直線即成功
  - R：判定框碰到距離右邊界 `checkDepth` pixel 的垂直線即成功
- UL、UR、DL、DR：判定目標是對應角落的兩條 `checkDepth` 檢定線交會處
  - UL：判定框需同時碰到上方與左方檢定線
  - UR：判定框需同時碰到上方與右方檢定線
  - DL：判定框需同時碰到下方與左方檢定線
  - DR：判定框需同時碰到下方與右方檢定線
- `checkDepth` 預設為 50 pixel，用來降低必須碰到真正邊框造成的操作難度

### 判定框（跟隨游標）
- 跟著滑鼠游標移動的長方形判定框，中心點對齊游標位置
- 預設尺寸與 `src/assets/suto400.gif` 相同（400x240），可透過設定調整長寬
- 大空心圓或檢定線提示開始顯示後，只要判定框碰到該方位的判定點 / 判定線即算成功判定（perfect）
- debug 模式下，在判定框四周畫出綠色線框

### 檢定提示圖示（縮圓 / 檢定線動畫）
- 8 個方位的檢查點各自獨立顯示提示動畫，依各自對應的拍子時機顯示
- UL、UR、DL、DR：維持現在的圓點設計，顯示一個白色半透明實心圓，外圈有一個大空心圓
- U、D、L、R：改成在上、下、左、右的 `checkDepth` 區塊上色，提示玩家要碰哪一邊；當前拍會顯示對應檢定線
- 提示顯示期間皆可判定；提示動畫結束前都沒有碰到才算失敗
- 縮圓動畫的提前顯示時間可設定（單位：毫秒），預設 1000ms
  - 例如設定 1000ms 代表在原本拍點前 1000ms 開始播放縮圓動畫
- perfect 跟 miss 文字顯示在判定框與判定點 / 判定線最接近或觸碰的位置

### 特殊檢定：旋轉
- 玩家根據提示階段的旋轉方向來進行檢定
- 每一拍會依序顯示兩個方向的縮圈檢查點
  - 例如：第一拍箭頭從 U 轉到 R，會先顯示 U，再顯示 UR；任一檢查點的大空心圓顯示後，判定框碰到即算該檢查點成功
  - 其餘七拍以此類推

### 游標顯示
- 檢定階段：`src/assets/suto400.gif`（400x240）的中心點顯示在滑鼠游標位置，持續循環播放
  - 成功判定時，疊加一個閃爍效果
- 提示階段：顯示靜態圖片 `src/assets/suto400.png`（400x240），同樣跟著滑鼠游標移動
- 離開檢定階段後隱藏 GIF

### 生命條
- 在左方顯示一條垂直的白色生命條，總值 100
  - perfect：+5 分
  - miss：-20 分
- 生命條顏色依分數變化：
  - 0–30：紅色
  - 31–70：黃色
  - 71–100：綠色
- 不顯示數值
- 生命歸零時立即觸發 Game Over，顯示「按任意鍵繼續」，玩家按鍵後回到主畫面

### BPM
- BPM 是可控參數，可在遊戲開始前設定，預設值 120

## 暫停系統
- 遊玩時按 ESC 暫停遊戲，顯示三個按鈕：
  - 繼續
  - 設定
  - 返回主頁
- 按「繼續」後，播放三拍預備拍並依序顯示倒數 3、2、1，倒數結束後恢復遊戲

## 關卡系統
- 難度從簡單到越來越難
- 每一個 stage 有 6 組（提示 + 檢定環節），共 6 × 8 × 2 拍
- 測試期間：一個 stage 先設定一組（提示 + 檢定環節），共 8 × 2 拍，之後再新增更多組

### HUD 顯示
- 畫面上方中間顯示當前 stage 編號（例如 `Stage 1`）
- stage 下方顯示當前輪次（例如 debug 時顯示 `1/1`，正式時顯示 `1/6`、`2/6` … 依此類推）

### 4x2 圖示排列順序
- 由左到右、由上到下對應拍序：beat 1 在左上、beat 8 在右下

## 關卡編輯格式（JSON）
```json
{
  "stages": [
    {
      "stage_number": 1,
      "bpm": 120,
      "sections": [
        {
          "type": "normal",
          "prompts": ["U", "D", "L", "R", "U", "D", "L", "R"]
        }
      ]
    },
    {
      "stage_number": 2,
      "bpm": 120,
      "sections": [
        {
          "type": "normal",
          "prompts": ["UL", "UR", "DL", "DR", "UL", "UR", "DL", "DR"]
        }
      ]
    },
    {
      "stage_number": 3,
      "bpm": 120,
      "sections": [
        {
          "type": "rotation",
          "start": "U",
          "rotate": ["R", "R", "R", "R", "R", "R", "R", "R"]
        }
      ]
    }
  ]
}
```
- `type: "normal"` 的 prompts：每個元素代表該拍要碰的檢定方位（第 1 拍→prompts[0]，第 2 拍→prompts[1]，以此類推）
- `type: "rotation"` 的欄位：
  - `start`：旋轉起始方向，只能是 `"U"`、`"D"`、`"L"`、`"R"` 其中之一
  - `rotate`：長度為 8 的陣列，每個元素代表該拍的旋轉方向，`"L"` 為逆時針，`"R"` 為順時針
  - 每拍的兩個檢查點由程式根據當拍的起始方向與 rotate[i] 自動推算，玩家需在兩個檢查點 / 檢定線提示顯示期間碰到它們

## 進入遊戲畫面
- 遊戲名稱（SUTO）
- 開始按鈕
- 設定（含 BPM、縮圈提前時間、判定框尺寸）

## 可設定參數彙整
| 參數 | 預設值 | 說明 |
|---|---|---|
| BPM | 120 | 每分鐘拍數 |
| shrinkLeadMs | 1000 | 縮圈動畫提前開始時間（毫秒） |
| hitboxWidth | 400 | 判定框寬度（像素） |
| hitboxHeight | 240 | 判定框高度（像素） |
| checkDepth | 50 | U、D、L、R 的檢定線距離畫面邊界的深度（像素）；斜角需同時碰到兩條對應檢定線 |
| debugMode | false | 開啟後顯示白色橢圓線與綠色判定框線框 |
